## 2. 이번 주 새로 추가된 기능

### ✨ 웨스트민스터 소교리문답 코너 (PR #27, #28)
- 107개 문답 전체를 앱에서 바로 볼 수 있습니다 (예장 합동 표준).
- "하나님 / 구원 / 십계명 / 기도" 4개 분류 탭으로 필터링 가능.
- **외우기 모드**: 답을 가리고 문제만 보여 준 뒤, 탭하면 정답이 나타납니다.
- **외움 표시**: 외운 문답에 체크 표시를 남기면 진도 막대(0/107)로 확인 가능.
- **소교리문답 퀴즈**: 10문제를 무작위로 뽑아 객관식으로 풀 수 있습니다.
  보기 섞기(Fisher-Yates 알고리즘)도 올바르게 적용되어 있습니다. 👍

### ✨ 목사님 현황판 (/admin) (PR #28)
- 관리자(목사님)만 접근 가능한 전용 페이지가 생겼습니다.
- 성도별 양육 과정 수료 진도(0/과목수)와 퀴즈 참여 횟수·획득 포인트를 한 화면에 볼 수 있습니다.
- 비관리자가 접근하면 "관리자 전용" 안내 화면을 보여 줍니다. 👍

### ✨ 교인 절차 안내 페이지 (/guide) (PR #26)
- 새신자·학습교인·세례교인·입교 과정을 단계별로 안내하는 고정 페이지가 추가되었습니다.

## 3. 발견된 문제 (우선순위 순)

---

### 🔴 A. 휴대폰 앱 아이콘 파일이 없음 *(지난주 이월·미해결)*
- **무슨 일?** 앱을 홈 화면에 설치할 때 필요한 아이콘 이미지(`icon-192.png`, `icon-512.png`)가
  저장소에 여전히 없습니다.
- **그래서 뭐가 문제?** 홈 화면에 설치 시 아이콘이 빈 칸이나 깨진 모양으로 표시됩니다.
- **권장:** 192×192·512×512 크기 이미지를 `public/icons/` 폴더에 추가.

### 🔴 B. 인터넷이 끊기면 "문제를 불러오는 중..." 화면에서 멈춤 *(지난주 이월·미해결)*
- **무슨 일?** 퀴즈 화면에서 네트워크 오류 발생 시 대기 화면에서 빠져나올 방법이 없습니다.
- **그래서 뭐가 문제?** 지하철·약한 와이파이 환경에서 앱이 먹통처럼 느껴집니다.
- **권장:** 일정 시간(예: 10초) 후 "다시 시도" 버튼과 안내 문구 표시.

### 🔴 C. 문제 개수 입력값에 제한이 없음 *(지난주 이월·미해결)*
- **무슨 일?** `/api/questions?count=999999`처럼 큰 값을 넣으면 데이터베이스에서
  그 5배 분량을 한꺼번에 조회합니다.
- **그래서 뭐가 문제?** 서버 과부하와 불필요한 비용이 발생할 수 있습니다.
- **권장:** 서버에서 count를 1~50 범위로 강제 제한.

---

### 🟡 D. 소교리문답 외움 기록이 기기마다 따로 저장됨 *(신규)*
- **무슨 일?** "외움 표시" 기능이 이 기기의 브라우저 저장소(`localStorage`)에만 저장됩니다.
  같은 사람이 스마트폰과 태블릿을 번갈아 사용하면 기록이 따로 놉니다.
  또한 브라우저 데이터를 지우거나 새 기기로 바꾸면 기록이 사라집니다.
- **그래서 뭐가 문제?** 성도가 꾸준히 외워 나가는 진도가 갑자기 초기화되는 불편함이 생깁니다.
  또한 목사님 현황판에서는 소교리문답 외움 진도가 전혀 보이지 않습니다.
- **권장:** 외움 기록을 데이터베이스(Supabase)에 저장해 기기 간 동기화. 현황판에도 반영.

### 🟡 E. 소교리문답 퀴즈 결과가 목사님 현황판에 반영되지 않음 *(신규)*
- **무슨 일?** 성경 퀴즈는 결과가 데이터베이스에 저장되어 현황판에 보이지만,
  소교리문답 퀴즈는 결과를 저장하지 않습니다.
- **그래서 뭐가 문제?** 성도가 소교리문답 퀴즈를 열심히 풀어도 목사님 현황판에는
  아무것도 표시되지 않아, 양육 현황을 파악하기 어렵습니다.
- **권장:** 소교리문답 퀴즈 결과를 별도 테이블(또는 기존 scores 테이블)에 저장하고 현황판에 표시.

### 🟡 F. 미사용 코드 정리 필요 *(지난주 이월)*
- Tailwind CSS가 설치만 되어 있고 사용하지 않습니다.
- `lib/supabase.ts`의 `supabaseAdmin`(강력한 권한의 DB 접속)이 만들어져 있지만 어디서도 쓰지 않습니다.
- `lib/bible.ts`의 `booksForTestament` 함수가 만들어져 있지만 화면에서 쓰지 않습니다.
- **권장:** 안 쓰는 코드는 제거 (특히 `supabaseAdmin`은 보안 위험 요소).

### 🟡 G. 문제 생성 실행 방법이 두 곳에서 다름 *(지난주 이월)*
- 사용 설명서(`package.json`)에는 `ts-node`로 실행하라고 되어 있지만,
  실제 자동 실행(GitHub Actions)은 `tsx`로 돌립니다.
- **권장:** 설명서의 실행 명령도 `tsx`로 통일.

---

### 🟢 H. 현황판: 성도 수가 늘면 느려질 수 있음 *(신규)*
- **무슨 일?** 목사님 현황판은 접속할 때마다 **모든 성도의 모든 퀴즈 기록**을
  한꺼번에 데이터베이스에서 불러옵니다.
  성도 수가 수백 명이 되고 각자 퀴즈를 많이 풀면 불러오는 데이터가 매우 커집니다.
- **그래서 뭐가 문제?** 현황판이 느리게 열리거나, 일부 데이터가 누락될 수 있습니다
  (Supabase는 기본적으로 한 번에 최대 1,000개 행만 반환).
- **권장:** 점수 합계는 DB 자체에서 미리 계산하도록 변경, 페이지 나누기 추가. (당장 급하지 않음)

### 🟢 I. 목사님 현황판 진입 경로가 없음 *(신규)*
- **무슨 일?** 현황판(`/admin`)으로 가는 버튼이나 메뉴가 홈 화면에 없습니다.
  URL을 직접 입력해야만 접근 가능합니다.
- **그래서 뭐가 문제?** 목사님이 현황판의 존재를 알고 있어야만 쓸 수 있습니다.
- **권장:** 관리자 계정으로 로그인한 경우 홈 화면 하단에 현황판 링크 표시.

### 🟢 J. 화면 설정의 구버전 경고 *(지난주 이월)*
- 앱 화면 기본 설정이 Next.js 14가 권장하지 않는 옛 방식으로 작성되어 있어
  빌드할 때마다 경고 메시지가 나옵니다.
- **권장:** 권장 방식(`viewport` 설정)으로 옮기기.

### 운영 측면 추가 제안
- **소교리문답 AI 퀴즈 문제 다양화**: 현재는 정답과 오답 보기를 모두 소교리문답 107문답에서 뽑습니다.
  보기가 신학 전문 용어로만 채워져 난이도가 매우 높습니다. 신학 초보자를 위한 쉬운 난이도 모드를 고려해 보세요.
- **목사님 관리자 지정 방법**: 현재는 데이터베이스 관리자가 직접 `is_admin` 값을 바꿔야 합니다.
  사용 설명서가 없으면 누구도 방법을 모릅니다. 간단한 가이드 문서나 관리자 이메일 목록 방식으로 단순화를 권장합니다.
- **지난주 AI 문제 품질 검수**: AI가 생성한 성경 퀴즈의 신학적 정확성은 사람이 확인하지 않습니다.
  출시 전 표본(10~20문항)을 목사님이 직접 검토하시길 권장합니다.

## 6. 잘 되어 있는 점
- **소교리문답 퀴즈 보기 섞기**가 올바른 방법(Fisher-Yates 알고리즘)으로 구현되었습니다.
  지난주에 보고한 개선 권고가 신기능에 바로 적용된 것입니다. 👍
- **목사님 현황판 보안**: 관리자가 아닌 사람은 자동으로 잠금 화면으로 안내합니다. 👍
- **소교리문답 외우기 모드**의 UX가 직관적입니다: 질문 영역을 탭하면 답이 나타나고,
  "외움 표시" 버튼을 눌러 진도를 기록할 수 있습니다. 👍
- **데이터베이스 보안 규칙(RLS)**이 켜져 있어 일반 사용자가 다른 사람의 개인 기록을 볼 수 없습니다. 👍
- **양육 과정·교인 절차 안내**가 계속 충실히 추가되어, 교회 양육 플랫폼으로서의 완성도가 높아지고 있습니다. 👍

- **홈 상단 버튼 개편**: 동행, 소그룹, 랭킹, 오답 노트 버튼 정리

## 3. 발견된 문제 (우선순위 순)

### 🔴 A. 번역·음성 API를 누구나 무제한으로 쓸 수 있음 — *비용 폭증 위험*

- **무슨 일?** 외국어 번역(`/api/translate`)과 음성 읽기(`/api/tts`) 기능이 **로그인 여부를 확인하지 않습니다.** 누구든 이 주소로 요청을 보내면 앱 개발사의 비용으로 번역·음성 변환이 실행됩니다. 인터넷에서 이런 취약점을 찾는 자동화 도구가 많아 발견 시 비용이 순식간에 불어날 수 있습니다.
- **그래서 뭐가 문제?** Google 번역 API 요금이 개발사 부담으로 청구됩니다. 대량 요청이 들어오면 예상치 못한 비용이 발생하고, 서비스가 느려질 수 있습니다.
- **권장:** 번역·음성 요청 전 로그인 여부 확인 추가, 그리고 IP 또는 사용자당 분당 요청 횟수 제한.

---

### 🔴 B. 나눔 모임 정원이 동시에 초과될 수 있음 — *운영 신뢰성*

- **무슨 일?** 나눔 모임에 참여할 때 정원(6명)을 확인하고, 자리가 있으면 가입합니다. 그런데 **여러 사람이 거의 동시에 마지막 한 자리를 눌렀을 때**, 모두 "자리 있음"으로 확인하고 동시에 가입해 버릴 수 있습니다. 결과적으로 6명이 넘는 사람이 같은 모임에 들어올 수 있습니다.
- **그래서 뭐가 문제?** 오프라인 소그룹 모임에서 정원 초과는 실제로 사람들이 공간을 찾지 못하는 문제로 이어집니다. 사용자 신뢰가 깎입니다.
- **권장:** 데이터베이스에서 정원 확인과 가입을 **하나의 원자적 동작**으로 처리 (예: 트랜잭션 또는 DB 트리거로 6명 초과 시 자동 거부).

---

### 🔴 C. 음성 읽기 서비스가 비공식 경로에 의존 — *서비스 중단 위험*

- **무슨 일?** 음성 읽기(TTS) 기능이 Microsoft의 **비공식** 음성 변환 주소를 사용합니다. Microsoft Edge 브라우저 내부에서만 쓰도록 만들어진 비공개 API를 우회 활용하는 방식입니다. 언제든 Microsoft 측에서 차단할 수 있습니다. (현재 Azure 공식 키도 같이 연동되어 있어, 키가 설정되어 있으면 공식 방식으로 먼저 실행됩니다.)
- **그래서 뭐가 문제?** 비공식 경로가 막히면(실제로 과거에 여러 번 막혔다가 열린 전례가 있습니다) 음성 읽기 기능 전체가 중단됩니다. 특히 라오스어 등 공식 지원이 없는 언어를 쓰는 사용자에게 영향이 큽니다.
- **권장:** Azure 공식 키가 설정되어 있는지 확인하고, 없다면 우선 발급. 비공식 경로는 보조(폴백) 수단으로만 유지하고 장기적으로 제거.

---

### 🟡 D. 개발자용 "권한 진단" 버튼이 일반 사용자에게도 보임 — *UX·정보 보안*

- **무슨 일?** 나눔 모임 페이지에서 일반 사용자(리더 권한이 없는 분)에게 **"🔧 권한 진단"** 버튼이 보입니다. 이 버튼을 누르면 내부 계정 상태, 데이터베이스 접속 여부 등 개발 정보가 화면에 표시됩니다.
- **그래서 뭐가 문제?** 비개발자 사용자가 혼란스러울 수 있고, 서버 내부 구조가 불필요하게 노출됩니다. 앱스토어 심사에서도 문제가 될 수 있습니다.
- **권장:** 진단 버튼을 관리자(`isAdmin`) 계정에게만 보이게 수정하거나, 정식 출시 전 제거.

---

### 🟡 E. 성경 읽기 본문이 아직 없음 — *핵심 기능 미완성*

- **무슨 일?** 메인 화면에 "성경 읽기" 메뉴가 있고 페이지도 만들어져 있지만, **실제 성경 본문 데이터의 라이선스가 아직 확보되지 않은 상태**입니다. 현재 사용자에게는 "허가 진행 중" 안내만 보입니다.
- **그래서 뭐가 문제?** 앱스토어에 "성경 읽기"를 메인 기능으로 내세웠을 때 심사 탈락이나 사용자 실망으로 이어질 수 있습니다.
- **권장:** 라이선스 획득 예상 일정을 정하고, 그 전에는 메인 화면에서 성경 읽기 버튼을 숨기거나 "준비 중" 뱃지를 명확히 표시.

---

### 🟡 F. 미얀마어 사용자는 음성 읽기를 들을 수 없음 — *다국어 지원 불완전*

- **무슨 일?** 다바르는 여러 언어를 지원하지만, **미얀마어(버마어)**는 Microsoft Azure 음성 변환 서비스에서 지원하지 않습니다. 미얀마어 사용자는 음성을 들을 수 없습니다.
- **그래서 뭐가 문제?** 미얀마어를 선택했는데 조용히 아무 소리도 안 나면, 사용자는 앱에 오류가 생겼다고 오해할 수 있습니다.
- **권장:** 미얀마어 선택 시 "이 언어는 음성을 지원하지 않습니다"라는 안내 문구를 UI에 표시.

---

### 🟡 G. 앱 설치 후 인터넷이 없으면 완전히 멈춤 — *오프라인 지원 미완성*

- **무슨 일?** 앱을 홈 화면에 설치(PWA)할 수 있지만, **인터넷 연결이 없을 때는 아무것도 동작하지 않습니다.** 서비스 워커(오프라인 캐싱 담당 코드)가 푸시 알림 수신 기능만 있고 페이지·데이터 저장 기능이 없습니다.
- **그래서 뭐가 문제?** "설치형 앱"처럼 보이지만 인터넷이 없으면 웹 앱과 같이 동작 불가. 특히 해외 선교지처럼 네트워크가 불안정한 환경의 사용자에게 불편합니다.
- **권장:** 자주 쓰는 성경 퀴즈 문제, 전도 자료 등을 기기에 저장해 오프라인에서도 쓸 수 있게 캐싱 전략 추가 (장기 과제).

## 4. 더 좋게 만들면 좋은 점들 🟢

1. **연속 풀기 기록이 1년 넘으면 부정확해질 수 있음:** 홈 화면의 연속 풀기(스트릭) 표시가 최근 400개 기록만 확인합니다. 매일 10문제씩 풀면 약 40일 분량이라 문제없지만, 하루 1문제씩 풀면 400일 후 오래된 기록이 잘려 숫자가 틀릴 수 있습니다. 기준을 일(日) 단위로 바꾸면 해결됩니다.

2. **랭킹 내 순위 계산이 느릴 수 있음:** "나의 순위"를 보여주기 위해 데이터베이스에 두 번 요청합니다 (상위 50명 목록 + 내 순위 별도 계산). 사용자가 늘수록 느려집니다. 데이터베이스 뷰에 순위 번호를 미리 계산해 두면 단번에 해결됩니다.

3. **번역 결과를 매번 새로 요청함:** 같은 문장을 반복해서 번역할 때도 매번 구글 서버에 요청합니다. 결과를 기기에 잠깐 저장해 두면 속도가 빨라지고 비용도 줄어듭니다.

4. **오류 발생 시 팀이 알 수 없음:** 사용자 기기에서 오류가 생겨도 개발팀이 자동으로 알림을 받지 못합니다. 에러 추적 서비스(Sentry 등)를 연결하면 버그를 빠르게 발견하고 대응할 수 있습니다.

5. **데이터베이스 구조 변경 기록이 없음:** 앱 기능이 늘어나면서 데이터베이스 구조도 바뀌어 왔지만, 변경 이력이 정리되어 있지 않습니다. 나중에 문제가 생겼을 때 이전 상태로 되돌리기 어렵습니다. 마이그레이션 파일을 관리하면 안전합니다.

| **2** | 나눔 모임 정원 동시 초과 방지 (이슈 B) | 실제 오프라인 모임 운영 신뢰성 | 중간 |

| **9** | 오프라인 지원 (이슈 G) | 선교지 등 네트워크 불안정 환경 대응 | 높음 |

### 운영 측면 추가 제안

- **앱스토어 심사 전 체크리스트:** 계정 삭제 기능, 개인정보 처리방침은 잘 준비되었습니다. 다만 "권한 진단" 버튼(이슈 D)과 성경 읽기 본문 부재(이슈 E)는 심사관이 지적할 수 있습니다.
- **비용 모니터링:** 번역·음성 API 비용이 매달 얼마나 나오는지 Google Cloud 콘솔에서 예산 알림을 설정해 두면 폭증 시 즉시 알 수 있습니다.
- **AI 생성 문제 품질 검수:** 자동으로 만들어진 성경 퀴즈 문제가 신학적으로 정확한지, 보기 순서나 표현이 자연스러운지 주기적으로 사람이 확인하는 절차를 권장합니다.

## 6. 참고 — 잘 되어 있는 점

- 지난 리포트에서 지적한 **"API 문제 수 제한 없음"** 이슈가 수정되어 최대 50개로 제한되어 있습니다. 👍
- **`.env.local` 파일이 git에 올라가지 않도록 설정**되어 있어 비밀키 유출 위험이 없습니다. 👍
- **번역 API는 서버에서만 Google 키를 사용**해 브라우저에 키가 노출되지 않습니다. 👍
- **프로필 컬럼 누락 시에도 앱이 정상 동작**하도록 단계별 폴백이 잘 구현되어 있습니다. 👍
- **나눔 모임 RLS(행 단위 보안 정책)** 기반 권한 관리로 데이터 접근 통제가 잘 되어 있습니다. 👍
- **흰 화면 자동 복구(ChunkGuard)** 기능이 추가되어 빌드 배포 후 캐시 문제로 하얀 화면이 뜨는 현상이 줄었습니다. 👍
